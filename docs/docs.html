Scope

Framework targets every form element in the document. However, layouts and presentation layers are triggered by class names.

Structure

Basic

Basic structure of the form looks like this (commented):

<form action="" method="post">
  <fieldset>
    <legend></legend>

  </fieldset>
</form>

The form is defined by <form> + <fieldset> elements. Inside there’s a <legend> element describing what the form does.

Form cell

<form action="" method="post">
  <fieldset class="{parameters}">
    <legend></legend>
    <div class="{type}">

    </div>
  </fieldset>
</form>

The hightlighted snippet defines the area for field and its label. It will be referenced as cell.

Class Names

  1. {parameters} is used to trigger visual changes in form and switch layout for cells globally.
  2. {type} declared the type of layout used for this specific cell.

Layout can be declared only in one place – use <fieldset> to apply one layout globally or declare it for each cell.

Inside cell

Code for almost all cells looks like this:

<div class="{type}">
  <label for="screenname">Screenname</label>
  <div class="field">
    <input type="text" id="screenname" />
  </div>
</div>

Here we use a single text field with corresponding label, bound together by for and id attributes.

Div element inside holds only one text field, so its classname is field. It will stretch field to 100% of container. To put a few fields near each other, use fields classname.

<div class="{type}">
  <label for="screenname">Screenname</label>
  <div class="fields">
    <input type="text" id="screenname" size="20" />
    <input type="text" id="email" size="15" />
  </div>
</div>

Layouts

Form layout can be organized in two ways – fields can be arranged to be displayed line after line or in a grid-like layout

Block

<fieldset class="block">
  <div>
    <label >Comment</label>
    <div class="field">
      <textarea rows="5" cols="40"></textarea>
    </div>
  </div>
  <div>
    <label>Name</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
  <div>
    <label>Email</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
</fieldset>

Grid

<fieldset class="grid">
  <div>
    <label class="noalign">Comment</label>
    <div class="field">
      <textarea rows="5" cols="40"></textarea>
    </div>
  </div>
  <div>
    <label>Name</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
  <div>
    <label>Email</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
</fieldset>

Default width of grid layout is split 25% for label, 75% for field(s). It’s possible to overwrite those values.

Mixed

In examples above, layout was specified by adding classname globally, to fieldset element. It’s possible to mix layouts by using class names for each cell.

<fieldset>
  <div class="block">
    <label class="noalign">Comment</label>
    <div class="field">
      <textarea rows="5" cols="40"></textarea>
    </div>
  </div>
  <div class="grid">
    <label>Name</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
  <div class="grid">
    <label>Email</label>
    <div class="field">
      <input type="text" />
    </div>
  </div>
</fieldset>

Visual style

Set

Legend

Outlines form with border and adds padding.

<fieldset class="set">
  <legend>Legend</legend>
  <p>Content</p>
</fieldset>

No legend

Hides legend from view, while allowing adding semantics to HTML.

<fieldset class="nolegend">
  <legend>Legend</legend>
</fieldset>

Separated block

Legend

Splits each cell with border and some margin.

Works only when layout is specified

<fieldset class="separated block">
  <legend>Legend</legend>
  <!-- form fields -->
</fieldset>

Separated grid

Legend

Splits each cell with border and some margin.

Works only when layout is specified

<fieldset class="separated grid">
  <legend>Legend</legend>
  <!-- form fields -->	
</fieldset>

No label

Legend

Works only when applied on a form with grid layout.

<fieldset class="separated grid">
  <legend>Legend</legend>
  <div class="nolabel">
    <div class="field">
      <input type="text" value="without label" />
    </div>
  </div>
</fieldset>
