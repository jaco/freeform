Scope

Framework targets every form element in the document. However, layouts and presentation layers are triggered by class names.

Structure

Basic

Basic structure of the form looks like this (commented):

<form action="" method="post">
  <fieldset>
    <legend></legend>

  </fieldset>
</form>

The form is defined by <form> + <fieldset> elements. Inside there’s a <legend> element describing what the form does.

Form cell

<form action="" method="post">
  <fieldset class="{parameters}">
    <legend></legend>
    <div class="{type}">

    </div>
  </fieldset>
</form>

The hightlighted snippet defines the area for field and its label. It will be referenced as cell.

Class Names

  1. {parameters} is used to trigger visual changes in form and switch layout for cells globally.
  2. {type} declared the type of layout used for this specific cell.

Layout can be declared only in one place – use <fieldset> to apply one layout globally or declare it for each cell.

Visual style

Set

Legend

Outlines form with border and adds padding.

<fieldset class="set">
  <legend>Legend</legend>
  <p>Content</p>
</fieldset>

No legend

Hides legend from view, while allowing adding semantics to HTML.

<fieldset class="nolegend">
  <legend>Legend</legend>
</fieldset>

Separated block

Legend

Splits each cell with border and some margin.

Works only when layout is specified

<fieldset class="separated block">
  <legend>Legend</legend>
  <!-- form fields -->
</fieldset>

Separated grid

Legend

Splits each cell with border and some margin.

Works only when layout is specified

<fieldset class="separated grid">
  <legend>Legend</legend>
  <!-- form fields -->	
</fieldset>

No label

Legend

Works only when applied on a form with grid layout.

<fieldset class="separated grid">
  <legend>Legend</legend>
  <div class="nolabel">
    <div class="field">
      <input type="text" value="without label" />
    </div>
  </div>
</fieldset>
